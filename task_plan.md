## 关键决策

- 以 LandPPT 作为产品外壳与流程编排层。
- 以 Linux.do 帖子的方法论作为需求澄清、研究、提纲与页规划层。
- 以 ppt-master 作为 SVG-first 渲染与 PPTX 导出内核。
- 当前对外产品定位统一收敛为“AI PPT 生成网站 / AI PPT Studio”，Web 端默认目标是产品主入口，而不是继续作为 thesis/dashboard 风格的内部开发页。
- 阶段 6 当前新增一段已完成的“多页面产品壳层”收口：在保持静态 HTML + 原生 ES module 技术路线不变的前提下，Web 入口已拆分为首页、项目列表页和单项目工作区页三类页面，项目跳转从 hash 锚点收敛为 query 参数路由，避免继续把产品工作流堆在单页长面板里。
- “论文答辩”保留为重点支持场景之一，但不再作为仓库、首页文案和阶段计划的默认产品身份；对外叙事统一覆盖答辩、汇报、咨询、培训、发布会、政务汇报、技术分享等多场景。
- 新系统同时支持对话生成与文件上传生成两种入口。
- 步骤 2 当前聚焦把 `apps/web` 首屏从“内部控制台观感”推进为真正的产品首页化入口，同时保留现有原生 ES module 结构与所有真实 API 绑定，不引入额外框架迁移。
